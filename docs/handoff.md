## 0. このリポジトリの目的

- `main` は「色々なアプリを作っていくための母艦リポジトリ」。
- 家族向け・個人向けの小さな静的Webアプリ（基本1つの `index.html` で完結）をためていく。
- スマホ（iPhone）で使う前提。PWA・モバイルファースト。

---

## 1. 確定した全体方針（重要）

### 2リポジトリ体制
- **`main`（Public／公開）** … アプリ公開用の母艦。無料の GitHub Pages で公開する。
  **全世界から見えるので、個人情報・機微情報はここに置かない。**
- **`private`（Private／非公開・別リポジトリ）** … 個人情報・プロフィール・開発メモを置く。
  Claude が参照するときはセッションのソースに追加して読む。

### 個人情報・データの扱い（この方針で合意済み）
- 個人の実データ（氏名・予定・住所など）は**コードに書かない**。
  → スマホの **`localStorage`** に保存する。これにより、アプリを Public 公開しても
  各自の実データは漏れない（データは各人の端末内だけ）。
- Claude 用のプロフィールは `private` リポジトリの `CLAUDE.md` に置く（`main` には置かない）。

### 家族への展開の考え方（ユーザーの疑問だった点）
- 家族は **GitHub を一切使わない**。公開された **URL を開くだけ**でアプリを使える。
  例: `https://hosoyama0909.github.io/main/apps/odekake/`
- iPhone なら「共有 → ホーム画面に追加」でアプリ化（PWA）。
- 無料 Pages の URL は「リンクを知っていれば誰でも開ける」（パスワードは無い）が、
  実データは各自の localStorage なので家族のデータは漏れない。

### リポジトリ構成の向き（検討して結論済み）
- 「main を非公開にして公開用リポを別に作る（逆パターン）」も検討したが、
  **今のまま（main=Public / private=Private）で進める**ことに決定。
  理由: アプリのコードには秘密が入らない設計なので、アプリを二重管理する必要がない。
