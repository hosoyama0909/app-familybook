# 用語集（Glossary）

| 用語 | 意味 |
|------|------|
| A-SPICE | Automotive SPICE。自動車ソフト向けのプロセス参照・アセスメントモデル。 |
| 作業成果物 (Work Product) | 各プロセスが生成する文書・コード・テスト等。 |
| トレーサビリティ | 要求〜テスト間の対応関係。上流・下流の双方向でたどれること。 |
| 一貫性 (Consistency) | トレースで結ばれた成果物同士の内容が矛盾しないこと。 |
| テーラリング | 標準プロセスをプロジェクト事情に合わせて調整すること。 |
| 能力レベル | プロセスの成熟度（0〜5）。 |
| CR (Change Request) | 変更依頼。 |

## 要求・設計IDの採番規則

| 接頭辞 | 種別 | 例 | 対応プロセス |
|--------|------|----|--------------|
| `STK-` | ステークホルダ要求 | STK-02 | SYS.1 |
| `SYR-` | システム要求 | SYR-10 | SYS.2 |
| `SWR-` | ソフトウェア要求 | SWR-LOG-02 | SWE.1 |
| `SWA-` | ソフトウェアアーキ要素 | SWA-LOGGER | SWE.2 |
| `SWD-` | 詳細設計要素 | SWD-recordMoment | SWE.3 |
| `UTC-` | 単体テストケース | UTC-AGE-01 | SWE.4 |
| `QTC-` | 適格性テストケース | QTC-LOG-01 | SWE.6 |
| `RSK-` | リスク | RSK-03 | MAN.5 |

機能ごとに中間キーを入れてよい（例：`SWR-LOG-*` = きろく機能, `SWR-CHILD-*` = お子さん登録）。
