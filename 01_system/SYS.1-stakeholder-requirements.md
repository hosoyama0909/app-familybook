# SYS.1 ステークホルダ要求

出典：壁打ち・ロードマップ（`private` リポジトリの `PLAN.md` / `roadmap.html`）から抽出。
ステークホルダ＝オーナー（開発者本人）＋利用者（子育て中の家族）。

| ID | ステークホルダ要求 | 由来 | 優先度 |
|----|--------------------|------|--------|
| STK-01 | 家族のおでかけを計画し、当日その場でナビして回りたい（＝旅アシスト） | 既存プロトタイプ | 高 |
| STK-02 | 親のリアルな育児イベント（グズり・抱っこ等）を、疲れていても残したい | 「グズりボタン」着想 | 高 |
| STK-03 | 記録が後で「物語」として家族で見返せるものになってほしい | 「思い出は写真より物語」 | 中 |
| STK-04 | 行った場所が地図に人生として積み重なってほしい | 「家族ヒストリーマップ」 | 中 |
| STK-05 | 育児イベントを次回の計画に活かしたい（休憩提案など） | 「特許の芽」 | 中 |
| STK-06 | ログイン不要・無料で、スマホですぐ使えること | 利用者体験 | 高 |
| STK-07 | 子どもの位置情報・写真を扱うため、プライバシーが守られること | 品質・倫理 | 高 |
| STK-08 | 10年後も見返したくなる、感情に寄り添うデザインであること | プロダクト哲学 | 中 |

## ロードマップとの対応（段階的実現）
- Stop 0（実装済み）＝ STK-01, STK-06
- **Stop 1（実装済み）＝ STK-02, STK-05の土台（子ども登録）**
- Stop 2 ＝ STK-03, STK-04
- Stop 3〜4 ＝ STK-03（AI物語）, STK-05
- 全体 ＝ STK-07, STK-08
