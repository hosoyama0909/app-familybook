# SYS.2 システム要求分析

STK 要求を、システム（アプリ）が満たすべき要求に分解する。各 SYR は検証可能な形で書く。

## 機能要求

| ID | システム要求 | 由来(STK) | 検証 |
|----|--------------|-----------|------|
| SYR-01 | 複数の「おでかけ」を作成・切替・削除できる | STK-01 | QTC/手動 |
| SYR-02 | 各おでかけに日ごとのスケジュールを登録・表示できる | STK-01 | 手動 |
| SYR-03 | 予定・お店・スポットから地図ナビ（外部地図）を開ける | STK-01 | 手動 |
| SYR-04 | 持ち物・予算・写真を管理できる | STK-01 | 手動 |
| SYR-05 | 次の予定までのカウントダウンと現在の状況を表示する | STK-01 | 手動 |
| **SYR-10** | **育児イベントをワンタップで時系列に記録できる** | STK-02 | QTC-LOG-01 |
| **SYR-11** | **記録に時刻を自動付与し、許可時は位置も付与する** | STK-02,05 | QTC-LOG-01 |
| **SYR-12** | **記録を当日サマリー・日別タイムラインで振り返れる** | STK-03 | QTC-LOG-01 |
| **SYR-13** | **お子さんの生年月を登録し、年齢を自動算出できる** | STK-05 | UTC-AGE-01 |
| **SYR-14** | **しおり（予定）と実績（きろく）を時系列統合した「ふりかえり」を表示する** | STK-03 | QTC-REC-01 |
| **SYR-15** | **おでかけの都道府県を記録し、日本地図上に訪問履歴（コンプリート）として可視化できる** | STK-04 | QTC-PREF-01 |
| **SYR-16** | **おでかけのデータからAIが物語文を生成し、保存・再表示できる** | STK-03 | QTC-STORY-01 |
| **SYR-17** | **お子さんごとに、その旅の「手紙」をAIが生成し保存・再表示できる** | STK-03 | QTC-LETTER-01 |
| **SYR-18** | **記録から親のがんばり度（親HP）を算出し表示する** | STK-05 | QTC-HP-01 |
| **SYR-19** | **複数の記録から家族の傾向（グズり時間帯・疲れやすい経過時間）を学習し、休憩の目安を提案する** | STK-05 | QTC-LEARN-01 |
| **SYR-21** | **都道府県の訪問偏りから、次の新規開拓先を提案する** | STK-04,05 | QTC-OPEN-01 |
| SYR-20 | データを書き出し／取り込みでき、家族と共有できる | STK-06 | 手動 |

## 非機能要求（品質特性）

| ID | 非機能要求 | 由来 | 検証 |
|----|-----------|------|------|
| SYR-N1 | ログイン不要・サーバ不要で動作する（クライアント完結） | STK-06 | アーキ審査 |
| SYR-N2 | データは端末内（localStorage）に保持する。**AI機能を使う時のみ**、最小化したデータを中継サーバ経由で送信する（下記N7）。それ以外は外部送信しない | STK-07 | コード審査 |
| **SYR-N7** | **AI送信は明示的な同意表示のうえで行い、送信内容を最小化する（GPS座標・写真・本名を送らない）** | STK-07 | QTC-STORY-02 |
| **SYR-N8** | **AI APIキーはクライアントに置かず、サーバ（Cloudflare Worker）側の秘匿情報として保持する** | STK-07 | アーキ審査 |
| **SYR-N9** | **AI中継サーバ未設定・通信失敗・オフライン時も、既存機能は継続動作する（劣化）** | STK-06 | QTC-STORY-04 |
| SYR-N3 | 位置情報の取得は利用者が拒否しても主要機能が継続する | STK-07 | QTC-LOG-02 |
| SYR-N4 | スマホ縦画面・ライト/ダーク両対応、片手で操作可能 | STK-06,08 | 目視 |
| SYR-N5 | PWAとしてホーム画面追加・オフライン閲覧に耐える | STK-06 | 手動 |
| SYR-N6 | 既存データ形式との後方互換（マイグレーション）を保つ | 品質 | UTC/手動 |

## 制約
- 単一 `index.html`（HTML/CSS/JS 一体）。外部ライブラリ非依存（現状）。
- ストレージは localStorage（容量制約 ~5MB、写真は圧縮して保存）。

トレースは `05_traceability/traceability-matrix.md` に集約。
