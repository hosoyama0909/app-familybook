# CLAUDE.md

このリポジトリで作業するときに Claude Code が最初に読む「取扱説明書」です。
コーディングルール・フォルダ構成・プロフィール情報をここに集約します。
迷ったらこのファイルのルールに従ってください。

---

## 1. このリポジトリは何か

**色々なアプリを作っていくための母艦リポジトリ**です。
家族向け・個人向けの小さなWebアプリをたくさんこのリポジトリにためていきます。

### リポジトリ構成（2リポジトリ体制）
- **`main`（このリポジトリ・Public／公開）** … アプリ公開用の母艦。無料のGitHub Pagesで公開する。**⚠️ 全世界から見えるので、個人情報・機微情報はここに置かない。**
- **`private`（別リポジトリ・Private／非公開）** … 個人情報・プロフィール・開発メモを置く場所。Claudeが参照するときはセッションに追加して読む。

### アプリの前提
- 各アプリは基本「1つのHTMLファイルで完結する静的Webアプリ」
- スマホ（iPhone）で使うことが多い → PWA・モバイル前提でデザインする
- **個人の実データ（氏名・予定・住所など）はコードに書かず、スマホの `localStorage` に保存する**（Publicでも実データが漏れない）

---

## 2. フォルダ構成のルール

```
main/
├── index.html          ← アプリ一覧ランチャー（全アプリの入口）
├── CLAUDE.md           ← このファイル
├── apps/               ← アプリはすべてここに1フォルダずつ
│   ├── odekake/        ← 例: 家族おでかけアプリ
│   │   └── index.html
│   └── _template/      ← 新規アプリの雛形（コピー元）
│       └── index.html
└── .github/workflows/  ← GitHub Pages公開などの自動化
```

### 命名ルール
- アプリのフォルダ名は **半角小文字英数字とハイフン**（例: `odekake`, `kakei-bo`, `todo-list`）
- 各アプリのメインファイルは必ず `apps/<name>/index.html`
- `_template/` などアンダースコア始まりは「アプリではない補助フォルダ」の目印

---

## 3. 新しいアプリを作るときの手順

1. `apps/_template/` をコピーして `apps/<新しいアプリ名>/` を作る
2. `apps/<新しいアプリ名>/index.html` を実装する
3. ルートの `index.html`（ランチャー）に新しいアプリのカードを追加する
4. コミットしてプッシュする

「新しいアプリを作って」と言われたら、この手順に沿って進めてください。

## 4. コーディングルール

既存の `apps/odekake/` のスタイルを踏襲します。

### 基本方針
- **1つの `index.html` で完結**させる（HTML・CSS・JS を1ファイルに内包）。外部ライブラリやCDNは原則使わない
- 画像・アイコンは可能な限り **絵文字 or インラインSVG（data URI）** で済ませる（外部ファイル依存を減らす）
- **PWA対応**: `<link rel="manifest">` をインラインdata URIで持たせ、`apple-mobile-web-app-*` メタタグを付ける
- **モバイルファースト**: iPhoneでの表示・タップ操作を最優先。`viewport-fit=cover` でセーフエリア対応

### CSS
- 色・フォントは `:root` の **CSS変数**で定義し、使い回す（ハードコードしない）
- ライト/ダーク両対応を意識（`data-theme` 属性で切替できる形が理想）
- 日本語UIなので明朝/ゴシックのフォント指定を用意する

### JavaScript
- バニラJS（フレームワークなし）。ビルド不要で「ファイルを開けば動く」状態を保つ
- データを保存する場合は `localStorage` を使う（サーバー不要）
- コメントは日本語でOK。処理の意図がわかる程度に簡潔に

### 品質
- スマホの実機幅（375px前後）で崩れないか意識する
- ボタンやリンクは指でタップしやすいサイズ（最低44px程度）にする

## 6. デプロイ（GitHub Pages）

- `.github/workflows/pages.yml` でリポジトリ全体をGitHub Pagesに公開します
- 公開URLの形: `https://hosoyama0909.github.io/main/apps/<アプリ名>/`
- `main` は Public なので**無料で公開できます**。事前に Settings > Pages で「Source: GitHub Actions」を選ぶこと
- ⚠️ Public公開なので、アプリのコードに個人の実データを書かない（実データは `localStorage` へ）
